# Frontend Prompt Refinement Checklist

Use this checklist when a prompt is too vague or when the user wants a production-quality frontend implementation prompt.

## Context
- Existing app/framework and version
- Target route/page/component
- User persona and primary task
- Design source: screenshot, Figma, design system, or freeform
- Browser/device support

## Functional Detail
- User inputs and validation rules
- Data source/API shape
- Permission/auth states
- State transitions
- Edge cases and failure states

## Frontend Quality
- Component boundaries
- State management location
- Responsive layout
- Accessibility and keyboard behavior
- Loading/empty/error states
- Performance constraints
- Test coverage expectations

## Deliverability
- Expected files or modules
- Commands to run
- Acceptance criteria
- Manual QA checklist
- Explicit non-goals
